## Perfiles de modo de revisiÃ³n

El ajuste `review_mode` controla la exigencia del reviewer al evaluar cÃ³digo:

### standard (por defecto)

- **Prioridad**: Seguridad &rarr; CorrecciÃ³n &rarr; Tests &rarr; Arquitectura &rarr; Estilo
- **Bloquean**: Problemas de seguridad, errores lÃ³gicos, tests rotos, sobreescritura de ficheros
- **No bloquean**: Sugerencias de estilo

### paranoid

- **FilosofÃ­a**: Todo cambio es sospechoso hasta que se demuestre seguro
- **Prioridad**: Seguridad &rarr; CorrecciÃ³n &rarr; Tests &rarr; Integridad de datos &rarr; Arquitectura &rarr; Estilo

- **Umbral de confianza**: Aprobar solo si confianza > 0.85. Default a RECHAZO

### strict

- **Prioridad**: Seguridad &rarr; CorrecciÃ³n &rarr; Tests &rarr; Arquitectura &rarr; Estilo
- **Bloquean**: Todos los problemas de seguridad, errores lÃ³gicos, cobertura de tests insuficiente, falta de manejo de errores para llamadas externas
- **Umbral de confianza**: Rechazar si confianza < 0.80

- **FilosofÃ­a**: Preferir aprobar con sugerencias
- **Prioridad**: Seguridad crÃ­tica &rarr; Bugs claros &rarr; Tests de rutas crÃ­ticas
- **Bloquean**: Solo seguridad crÃ­tica (secrets, SQL injection, XSS) y errores lÃ³gicos claros

### custom
